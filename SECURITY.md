# Security Policy

Report security issues privately by emailing support@persistly.app.

Do not open public GitHub issues for vulnerabilities, leaked keys, authentication bypasses, cross-project save access, provider token exposure, or runtime API abuse paths.

Include:

- affected Phaser adapter version
- affected `@persistlyapp/sdk` version, if known
- affected runtime API endpoint, if relevant
- reproduction steps
- impact summary
- whether any runtime keys, provider tokens, account sessions, or project data were exposed
